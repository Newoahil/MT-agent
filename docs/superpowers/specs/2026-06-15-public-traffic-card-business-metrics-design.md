# 公域日报卡片经营指标设计

## 背景

当前公域日报飞书卡片展示了较多原始订单和履约指标，包括创建订单、签约订单、审出订单、发货订单、待发货、归还、逾期、关单，以及履约比率中的 `审出/签约`。实际使用中，`审出`相关数据长期为空或价值有限，占用了卡片空间。

业务上更需要日报卡片直接呈现二次加工后的经营指标，尤其是关单率。短期目标是将关单率控制在 35% 以内，这是进入活动阵地的门槛。

## 目标

- 从飞书卡片中移除低价值的 `审出订单` 和 `审出/签约` 展示。
- 将履约原始指标收窄为一行展示。
- 新增经营指标行：发货率、关单率、客单价。
- 将关单率目标线 `<=35%` 明确展示在卡片中。
- 关单率超过 35% 时给出风险提示；达标时给出达标提示。

## 非目标

- 不改变订单分析抓取逻辑。
- 不改变 Excel 订单分析 sheet 原始字段。
- 不改变日报上下文 JSON 中保存的原始订单分析数据。
- 不新增活动阵地自动判断或执行动作。

## 指标定义

### 发货率

`发货率 = 发货订单数 / 创建订单数`

用途：衡量从创建到实际发货的有效履约转化。

### 关单率

`关单率 = 关单数 / 创建订单数`

目标：`<=35%`

展示规则：

- `<=35%`：显示为达标。
- `>35%`：显示为风险。
- 分母缺失或为 0：显示 `-`，不判定达标/风险。

### 客单价

`客单价 = 签约金额 / 签约订单数`

用途：比单独展示签约金额更适合快速判断订单质量。

## 飞书卡片结构

订单和履约区域调整为三块：

1. 公域基础
   - 曝光
   - 公域访问
   - 商品页访问

2. 订单经营
   - 创建订单
   - 签约订单
   - 发货订单
   - 签约金额

3. 经营指标
   - 发货率
   - 关单率（含 `目标<=35%` 和达标/风险提示）
   - 客单价

4. 履约一行
   - 待发货
   - 归还
   - 逾期
   - 关单

其中履约一行应收窄为单行，不再拆成多行卡片；若飞书组件宽度不足，保持同一 `column_set` 内横向排列，由飞书客户端自行换行。

## Markdown 和文本输出

Markdown 日报和飞书 fallback 文本应与卡片口径保持一致：

- 移除 `审出订单` 和 `审出/签约` 的显著展示。
- 新增发货率、关单率、客单价。
- 保留必要原始数，便于追溯：创建订单、签约订单、发货订单、关单数。

## 数据来源

继续使用 `context.orderAnalysis`：

- `overview` 页：创建订单数、签约订单数、发货订单数、签约完成金额。
- `customs` 页：关单数。
- `delivery` 页：待发货订单数。
- `return` 页：归还订单数、逾期订单数。

如果某项缺失，展示 `-`，不阻断卡片生成。

## 测试要求

- 卡片不再包含 `审出订单`。
- 履约比率不再包含 `审出/签约`。
- 卡片包含 `发货率`、`关单率`、`客单价`。
- 关单率使用 `关单数 / 创建订单数`。
- 关单率超过 35% 时包含风险提示。
- 关单率小于等于 35% 时包含达标提示。
- Markdown/fallback 文本与卡片使用一致的经营指标口径。
